Using roslaunch

Posted by Erick Vieyra on Thu 21 September 2017

As seen on other posts in these blog, there are several things that need to be set-up and executed separately in order to have a functioning robot system, for example: execute several nodes, set parameters, start the master ROS node, etc. Roslaunch is a ROS tool that greatly simplifies this process by allowing us to execute several nodes at once, pass arguments to them or to the parameter server, or even employing simple conditionals to control the way in which our system starts!

Using roslaunch

ROS provides a command-line tool called roslaunch. The general usage is roslaunch [filename] and roslaunch [package] [filename]. We can also specify parameters and other options in the command line. We will explore some of these cases here.

Creating a launch file

Roslaunch takes a specific type of file as its parameter. Launch files are special XML-based files that contain information about what should roslaunch do whenever it is called by the user.

The general structure of a launch file is something like this:

<launch>
    <node name='node_1' />
</launch>
