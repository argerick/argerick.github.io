Installing ROS Kinetic Kame

Posted by Erick Vieyra on Mon 27 February 2017

This post has a quick and simple way of installing ROS Kinetic Kame and configuring a development environment in Ubuntu 16.04. Other operating systems, as well as other versions of ROS can be found in the ROS installation guide.

Installation

Follow the next steps to install ROS Kinetic Kame in Ubuntu 16.04. Keep in mind that Kinetic Kame is only supported in Ubuntu 15.10 and 16.04!

Adding the repositories

We will start by adding the repositories from packages.ros.org as a software source for Ubuntu.

sudo sh -c 'echo "deb http://packages.ros.org/ros/ubuntu $(lsb_release -sc) main" > /etc/apt/sources.list.d/ros-latest.list'

The keys for verifying the repositories are added with the next command:

sudo apt-key adv --keyserver hkp://ha.pool.sks-keyservers.net:80 --recv-key 421C365BD9FF1F717815A3895523BAEEB01FA116

Installing ROS Kinetic Kame

Now we update the software list of our system.

sudo apt-get update

... and we start a complete installation of ROS (base + rqt, rviz, robot-generic libraries, 2D/3D simulators, navigation and 2D/3D perception).

sudo apt-get install ros-kinetic-desktop-full

For other installation options, visit the official ROS installation guide.

Preparing a development environment

We will add some tools and utilities that allow us to easily manage our workspace.

Rosdep and Rosinstall

Rosdep is a tool that automatically solves dependencies needed by other ROS packages in our system.

The following commands start and update this tool in order to ensure that our workspace works correctly.

sudo rosdep init
rosdep update

Rosinstall is a Python tool that downloads and installs ROS packages. We can add this utility with the following command:

sudo apt-get install python-rosinstall

Configuring the environment

We will setup the environment variables that Linux needs to be able to use ROS commands from the terminal.

Running the bash script

ROS has a bash script that, when executed, configures the necessary variables for our environment. We can add a source command in the .bashrc file that loads this script thus easily setting up our bash sessions. .bashrc is by default the file that runs every time we open a new terminal in Ubuntu. To add our code at the end of the file run the following command:

echo "source /opt/ros/lunar/setup.bash" >> ~/.bashrc

... and reload for the current bash session:

source ~/.bashrc

Creating a workspace

We will now create a Catkin workspace (which is the tool that ROS uses to compile its source files) and we will add said workspace to our default bash session so we can execute our packages in a simple way.

Creating a Catkin workspace

Using the standard Linux command for creating and navigating directories, lets create a directory called catkin_ws and a subdirectory catkin_ws/src. We will then navigate to the subdirectory and initialize our workspace.

mkdir -p ~/catkin_ws/src
cd ~/catkin_ws/src
catkin_init_workspace

"Adding" our workspace to the terminal

We can run a source command in our .bashrc file to export our workspaces variables and settings. We will accomplish this exporting the setup.bash to the bash session:

echo "source catkin_ws/devel/setup.bash" >> ~/.bashrc
source ~/.bashrc

catkin_make

To compile the ROS packages in our workspace, we can run the catkin_make command from the root of the workspace:

catkin_make

Once the workspace has been compiled, we can start using its packages!
