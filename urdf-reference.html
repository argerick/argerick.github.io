URDF Reference

Posted by Erick Vieyra on Thu 21 September 2017

Universal Robot Description File (or URDF for short) is a file format used by ROS to describe a robot system using simple XML elements. This post serves as a reference to the most common elements used in this format and explains how they interact with each other.

Basic document structure

An URDF file is an XML document that follows a simple semantic to describe a robotic system. In its most basic form, an URDF file contains one or more <link> elements, and the corresponding <joint> elements that connect them.

The file has a <robot> element as its root. Inside this element the file contains several joints and links that compose a robot.

<robot>

    <link> ... </link>
    <link> ... </link>

    <joint> ... </joint>
    <joint> ... </joint>

</robot>

<robot>

The root element of every URDF file.

Attributes: - name (required) User-defined string representing the name of the robot.

Child elements: - < link > Describes a rigid body representing a link in the robot's kinematic chain.

  • < joint > Connections between links are specified with this element.

The <robot> element serves as the root of the URDF file. This element needs to have an attribute name specifying the name of the robot. Even though <link> and <joint> elements are not required to generate an URDF file, these elements are fundamental building blocks for any URDF file and skipping them will yield an 'empty' robot.

<link>

A rigid body representing a segment of a kinematic chain.

Attributes: - name (required) The unique name of the link. Other elements can refer to a link with this attribute.

Child elements: - < visual > Visual representation of the link. Can be specified as a geometry or import an external 3D file.

  • < inertial > Defines an inertial frame of reference for simulation purposes.

  • < collision > Collision geometry used for the link. Can be defined as a geometry or exported from a 3D file.

Link elements enclose three types of information: A visual representation, inertial data, and collision information. Keep in mind that these elements are optional and a link can just be defined without any sort of data associated with it (aside from its name).

<joint>

An element representing a connection between links

Attributes: - name (required) The unique name of the joint.

  • type (required) One of the six supported types of joint:
    • revolute: rotates along its axis within upper and lower limits.
    • continuous: rotates around its axis with no limits.
    • prismatic: slides along its axis within upper and lower limits.
    • fixed: all degrees of freedom are locked.
    • floating: moves freely along all six degrees of freedom.
    • planar: moves freely along a plane perpendicular to the axis.

Child elements: - < parent > (required) Specifies the name of the link that this joint belongs to.

  • < child > (required) Specifies the name of the link that belongs to this joint.

  • < limit > (required for revolute and prismatic joints) The upper and lower limits that revolute and prismatic joints must follow.

  • < origin > Transform between the parent link's frame and the joint's frame. Specified in xyz coordinates and roll-pitch-yaw (rpy) angles.

  • < axis > (default: (1 0 0)) The XYZ coordinates specifying the axis of the joint. The vector should be normalized.

  • < calibration > Reference positions for calibration purposes.

  • < dynamics > Damping and Friction coefficients for the joint.

  • < mimic > Specifies if the joint should mimic the movement of another joint.

  • < safety_controller > Position and speed-related information for limiting the joint's movement.

Joints are used to 'connect' several links together. This is done via the parent-child relationship that can be specified inside a <joint> element. <joint> elements must be specified with two attributes: name, which is just a string that identifies the joint; and type, which can be either "revolute", "continuous", "prismatic", "fixed", "floating" or "planar". It is mandatory to use the <limit> element if the joint is of type revolute or prismatic.

Customizing a < link > element

<visual>

A visual representation of a link. Several <visual> elements can be declared for a single link.

Attributes: - name string, name of this <visual> element.

Elements:

  • < origin > (default: link's frame) Transform between the link's frame and its visual representation.

    • xyz (default: (0 0 0) ) vector(3), contains x, y and z coordinates.
    • rpy (default: (0 0 0) ) vector(3), contains roll-pitch-yaw angles.

  • < geometry > Specifies the 3D representation to use.

    • < box > A rectangular prism which its origin in the center.
      • size vector(3), length of each side of the prism.
    • < cylinder > A cylinder defined by its radius and length with its origin in the center.
      • radius float, represents the radius.
      • length float, represents the length.
    • < sphere > A sphere defined by its radius and with its origin in the center.
      • radius float, represents the radius
    • < mesh > External file containing a 3D representation of the link. Both .dae (preferred) and .stl files are supported.
      • filename Path to the mesh file.
      • scale (default: 1) float, scaling factor for the mesh.
  • < material > Specifies the color or the texture that the link will display.
    • name string, used to reference the material within the link.
    • < color > The color of the material specified by its RGB and Alpha components.
      • rgba vector(4), red, green, blue and alpha values. Each in the range [0,1].
    • < texture > A file containing a texture for the link.
      • filename Path to a .jpg, .png or .bmp file.

<inertial>

An element containing information about the inertial properties of the link.

Attributes: none

Elements:

  • < origin > (default: link's frame) Transform between the link's frame and its inertial reference frame.

    • xyz (default: (0 0 0) ) vector(3), contains x, y and z coordinates.
    • rpy (default: (0 0 0) ) vector(3), contains roll-pitch-yaw angles.

  • < mass > The mass of the link in g.

    • value float, mass of the link.

  • < inertia > A \(3 \times 3\) rotational inertia matrix. The matrix can be defined with only 6 attributes since its symmetrical.

    • ixx, ixy, ixz, iyy, iyz, izz float, rotational inertia matrix element.

<collision>

An element containing information about the collision properties of the link. This element contains the same attributes/children as <visual>.

Attributes: - name string, name of this <visual> element.

Elements:

  • < origin > (default: link's frame) Transform between the link's frame and its visual representation. Same element structure as < visual >!
  • < geometry > Specifies a 3D collision model. Same element structure as < visual >!

Customizing a < joint > element

<parent>

Specifies the name of the link that this joint belongs to.

Attributes: - link string, name of the parent link

Elements: none

<child>

Specifies the name of the link that this joint belongs to.

Attributes: - link string, name of the child link

Elements: none

<origin>

Transform between the parent link's frame and the joint's frame.

Attributes: - xyz vector(3), contains x, y and z coordinates. - rpy vector(3), contains roll-pitch-yaw angles.

Elements: none

<limit>

The upper and lower limits that revolute and prismatic joints must follow.

Attributes: - lower (default: 0) float, lower joint limit (radians for revolute, meters for prismatic).

  • upper (default: 0) float, upper joint limit (radians for revolute, meters for prismatic).

  • effort (required) float, maximum joint effort.

  • velocity (required) float, maximum joint velocity.

Elements: none

<axis>

The XYZ coordinates specifying the axis of the joint.

Attributes: - xyz vector(3), contains x, y and z coordinates. Should be normalized.

Elements: none

<calibration>

Reference positions for calibrating the absolute position of the joint.

Attributes:

  • rising float, this reference triggers a rising edge with positive-direction movement.

  • falling float, this reference triggers a falling edge with positive-direction movement.

Elements: none

<dynamics>

Damping and Friction coefficients for the joint.

Attributes: - damping (default: 0) float, damping value given by \({N * m* s} / {rad}\) (revolute) or \({N * s} / m\) (prismatic).

  • friction float, static friction given by \(N\) (revolute) or \(N * m\) (prismatic).

Elements: none

<mimic>

Specifies if the joint should mimic the movement of another joint.

Attributes: - joint (required)

  • multiplier float, multiplying factor for the joint movement.

  • offset float, offset of the joint in radians or meters. A vector for joints that support it.

Elements: none

<safety_controller>

Specifies velocity and effort limits enforced by the robot controller. More info in safety limits.

Attributes: - soft_lower_limit (default: 0) float, lower joint boundary to start limiting the position

  • soft_upper_limit (default: 0) float, upper joint boundary to start limiting the position

  • k_position (default: 0) float, relation between position and velocity limits.

  • k_velocity (required) float, relation between effort and velocity limits.
